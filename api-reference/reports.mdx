## Create Report

Generate a shareable report from crawl results.

<ParamField header="Authorization" type="string" required>
  Bearer token with your API key
</ParamField>

<ParamField body="crawlId" type="string" required>
  The UUID of the completed crawl
</ParamField>

<ParamField body="type" type="string" default="summary">
  Report type: `summary`, `detailed`, `executive`
</ParamField>

<ParamField body="format" type="string" default="web">
  Output format: `web`, `pdf`
</ParamField>

<ParamField body="settings" type="object">
  Report configuration
  
  <Expandable title="Settings">
    <ParamField body="include_sections" type="array">
      Sections to include: `summary`, `broken_links`, `redirects`, `seo`, `performance`
    </ParamField>
    <ParamField body="password" type="string">
      Optional password protection
    </ParamField>
    <ParamField body="expires_in" type="integer">
      Days until link expires (null for never)
    </ParamField>
    <ParamField body="white_label" type="boolean" default="true">
      Apply white-label branding (Agency only)
    </ParamField>
  </Expandable>
</ParamField>

### Request

```bash
curl -X POST \
  "https://api.seocrawler.app/v1/reports" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "crawlId": "456e4567-e89b-12d3-a456-426614174100",
    "type": "detailed",
    "format": "web",
    "settings": {
      "include_sections": ["summary", "broken_links", "seo"],
      "password": "client-secret-123",
      "expires_in": 30,
      "white_label": true
    }
  }'
```

### Response

<ResponseField name="reportId" type="string">
  UUID of the generated report
</ResponseField>

<ResponseField name="url" type="string">
  Shareable URL for the report
</ResponseField>

<ResponseField name="format" type="string">
  Report format (web or pdf)
</ResponseField>

<ResponseField name="password_protected" type="boolean">
  Whether report requires password
</ResponseField>

<ResponseField name="expires_at" type="string">
  ISO 8601 expiration timestamp (null if never)
</ResponseField>

<ResponseExample>
```json
{
  "reportId": "report-123e4567-e89b-12d3-a456-426614174300",
  "crawlId": "456e4567-e89b-12d3-a456-426614174100",
  "domain": "example.com",
  "type": "detailed",
  "format": "web",
  "url": "https://reports.youragency.com/share/abc123xyz",
  "password_protected": true,
  "expires_at": "2025-01-01T00:00:00Z",
  "created_at": "2024-12-02T16:00:00Z"
}
```
</ResponseExample>

## Generate PDF Report

Request a PDF version of the report:

```bash

  "https://api.seocrawler.app/v1/reports" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "crawlId": "456e4567-e89b-12d3-a456-426614174100",
    "type": "executive",
    "format": "pdf"
  }'
```

  "format": "pdf",
  "status": "generating",

## Update Report

Update report settings:

```
PATCH /v1/reports/{reportId}
```

<ParamField body="password" type="string">
  Set or update password (null to remove)
</ParamField>

<ParamField body="expires_in" type="integer">
  Days from now until expiration (null for never)
</ParamField>

### Request

```bash
curl -X PATCH \
  "https://api.seocrawler.app/v1/reports/report-123e4567-e89b-12d3-a456-426614174300" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "password": null,
    "expires_in": 90
  }'
```

### Response

```json
{
  "reportId": "report-123e4567-e89b-12d3-a456-426614174300",
  "password_protected": false,
  "expires_at": "2025-03-02T16:00:00Z",
  "updated_at": "2024-12-02T17:00:00Z"
}
```

## Delete Report

Remove a report and invalidate its share link:

```
DELETE /v1/reports/{reportId}
```

### Request

```bash
curl -X DELETE \
  "https://api.seocrawler.app/v1/reports/report-123e4567-e89b-12d3-a456-426614174300" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

### Response

```json
{
  "deleted": true,
  "reportId": "report-123e4567-e89b-12d3-a456-426614174300"
}
```

## Report Types

| Type | Description | Pages |
|------|-------------|-------|
| `executive` | High-level summary for stakeholders | 1-2 |
| `summary` | Overview with key metrics | 3-5 |
| `detailed` | Full analysis with all issues | 10-20+ |

## Error Responses

| Status | Code | Description |
|--------|------|-------------|
| `400` | `crawl_not_complete` | Crawl must be completed first |
| `404` | `crawl_not_found` | Crawl doesn't exist |
| `404` | `report_not_found` | Report doesn't exist |
| `403` | `pdf_not_available` | PDF export not available on plan |
