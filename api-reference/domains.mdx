## List Domains

Retrieve all domains in your account.

<ParamField header="Authorization" type="string" required>
  Bearer token with your API key
</ParamField>

<ParamField query="status" type="string">
  Filter by verification status: `verified`, `pending`, `failed`
</ParamField>

<ParamField query="page" type="integer" default="1">
  Page number for pagination
</ParamField>

<ParamField query="limit" type="integer" default="20">
  Number of results per page (max: 100)
</ParamField>

### Request

```bash
curl -X GET \
  "https://api.seocrawler.app/v1/domains?status=verified&limit=10" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

### Response

<ResponseField name="data" type="array">
  Array of domain objects
  
  <Expandable title="Domain Object">
    <ResponseField name="id" type="string">
      Unique domain identifier (UUID)
    </ResponseField>
    <ResponseField name="domain" type="string">
      The domain name
    </ResponseField>
    <ResponseField name="status" type="string">
      Verification status: `verified`, `pending`, `failed`
    </ResponseField>
    <ResponseField name="health_score" type="integer">
      Latest health score (0-100), null if never crawled
    </ResponseField>
    <ResponseField name="last_crawl" type="string">
      ISO 8601 timestamp of last crawl
    </ResponseField>
    <ResponseField name="created_at" type="string">
      ISO 8601 timestamp when domain was added
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="pagination" type="object">
  Pagination metadata
  
  <Expandable title="Pagination">
    <ResponseField name="page" type="integer">
      Current page number
    </ResponseField>
    <ResponseField name="limit" type="integer">
      Results per page
    </ResponseField>
    <ResponseField name="total" type="integer">
      Total number of results
    </ResponseField>
    <ResponseField name="pages" type="integer">
      Total number of pages
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseExample>
```json
{
  "data": [
    {
      "id": "123e4567-e89b-12d3-a456-426614174000",
      "domain": "example.com",
      "status": "verified",
      "health_score": 85,
      "last_crawl": "2024-12-02T14:30:00Z",
      "created_at": "2024-11-01T10:00:00Z"
    },
    {
      "id": "223e4567-e89b-12d3-a456-426614174001",
      "domain": "blog.example.com",
      "status": "verified",
      "health_score": 92,
      "last_crawl": "2024-12-01T09:15:00Z",
      "created_at": "2024-11-05T14:22:00Z"
    }
  ],
  "pagination": {
    "page": 1,
    "limit": 10,
    "total": 2,
    "pages": 1
  }
}
```
</ResponseExample>

## Add Domain

Add a new domain to your account.

```
POST /v1/domains
```

<ParamField body="domain" type="string" required>
  The domain name to add (without protocol)
</ParamField>

<ParamField body="settings" type="object">
  Optional domain configuration
  
  <Expandable title="Settings">
    <ParamField body="protocol" type="string" default="auto">
      Protocol to use: `auto`, `https`, `http`
    </ParamField>
    <ParamField body="start_url" type="string" default="/">
      Starting URL path for crawls
    </ParamField>
    <ParamField body="rate_limit" type="integer" default="5">
      Max requests per second
    </ParamField>
  </Expandable>
</ParamField>

### Request

```bash
curl -X POST \
  "https://api.seocrawler.app/v1/domains" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "domain": "newsite.com",
    "settings": {
      "protocol": "https",
      "start_url": "/",
      "rate_limit": 3
    }
  }'
```

### Response

<ResponseExample>
```json
{
  "id": "323e4567-e89b-12d3-a456-426614174002",
  "domain": "newsite.com",
  "status": "pending",
  "verification": {
    "method": "dns",
    "token": "seocrawler-verification=abc123xyz789",
    "instructions": "Add TXT record: _seocrawler-verify.newsite.com"
  },
  "created_at": "2024-12-02T15:00:00Z"
}
```
</ResponseExample>

## Verify Domain

Trigger verification check for a pending domain.

```
POST /v1/domains/{domainId}/verify
```

<ParamField path="domainId" type="string" required>
  The UUID of the domain to verify
</ParamField>

### Request

```bash
curl -X POST \
  "https://api.seocrawler.app/v1/domains/323e4567-e89b-12d3-a456-426614174002/verify" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

### Response

<ResponseExample>
```json
{
  "id": "323e4567-e89b-12d3-a456-426614174002",
  "domain": "newsite.com",
  "status": "verified",
  "verification": {
    "method": "dns",
    "verified_at": "2024-12-02T15:05:00Z"
  }
}
```
</ResponseExample>

**Verification Failed Response:**

```json
{
  "id": "323e4567-e89b-12d3-a456-426614174002",
  "domain": "newsite.com",
  "status": "pending",
  "verification": {
    "method": "dns",
    "error": "TXT record not found",
    "last_check": "2024-12-02T15:05:00Z"
  }
}
```

## Delete Domain

Remove a domain from your account.

```
DELETE /v1/domains/{domainId}
```

<ParamField path="domainId" type="string" required>
  The UUID of the domain to delete
</ParamField>

### Request

```bash
curl -X DELETE \
  "https://api.seocrawler.app/v1/domains/123e4567-e89b-12d3-a456-426614174000" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

### Response

```json
{
  "deleted": true,
  "id": "123e4567-e89b-12d3-a456-426614174000"
}
```

<Warning>
  Deleting a domain permanently removes all associated crawl data and history. This action cannot be undone.
</Warning>

## Error Responses

| Status | Code | Description |
|--------|------|-------------|
| `400` | `invalid_domain` | Domain format is invalid |
| `404` | `domain_not_found` | Domain doesn't exist |
| `409` | `domain_exists` | Domain already in account |
| `422` | `domain_limit_reached` | Plan domain limit exceeded |
