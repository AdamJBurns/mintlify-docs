## Generate AI Review

Request an AI-powered analysis of a completed crawl.

<Info>
  **Available on**: Pro and Agency plans only. Uses your monthly AI Review allocation.
</Info>

<ParamField header="Authorization" type="string" required>
  Bearer token with your API key
</ParamField>

<ParamField path="crawlId" type="string" required>
  The UUID of the completed crawl to analyze
</ParamField>

<ParamField body="focus" type="array">
  Optional array of areas to focus the analysis
  
  Valid values: `broken_links`, `seo`, `performance`, `security`, `redirects`
</ParamField>

<ParamField body="context" type="string">
  Optional context about the site (e.g., "e-commerce site", "company blog")
</ParamField>

### Request

```bash
curl -X POST \
  "https://api.linkhealthmonitor.com/v1/crawl/456e4567-e89b-12d3-a456-426614174100/ai-review" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "focus": ["seo", "broken_links"],
    "context": "E-commerce website selling electronics"
  }'
```

### Response

<ResponseField name="reviewId" type="string">
  UUID of the AI review
</ResponseField>

<ResponseField name="status" type="string">
  Status: `processing`, `completed`, `failed`
</ResponseField>

<ResponseField name="health_score" type="integer">
  Overall health score (0-100)
</ResponseField>

<ResponseField name="summary" type="string">
  Executive summary of findings
</ResponseField>

<ResponseField name="critical_issues" type="array">
  High-priority problems requiring immediate attention
</ResponseField>

<ResponseField name="quick_wins" type="array">
  Easy fixes with significant impact
</ResponseField>

<ResponseField name="recommendations" type="array">
  Detailed recommendations with code fixes
</ResponseField>

## Check AI Review Status

AI Reviews may take 30-60 seconds to generate. Check status:

```
GET /v1/ai-review/{reviewId}
```

### Request

```bash
curl -X GET \
  "https://api.linkhealthmonitor.com/v1/ai-review/789e4567-e89b-12d3-a456-426614174200" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

### Processing Response

```json
{
  "reviewId": "789e4567-e89b-12d3-a456-426614174200",
  "status": "processing",
  "progress": 65,
  "estimated_remaining_seconds": 20
}
```

## AI Review Allocation

Check your remaining AI reviews:

```
GET /v1/account/usage
```

```json
{
  "ai_reviews": {
    "used": 3,
    "limit": 5,
    "resets_at": "2024-12-15T00:00:00Z"
  }
}
```

## Error Responses

| Status | Code | Description |
|--------|------|-------------|
| `400` | `crawl_not_complete` | Crawl must be completed before review |
| `403` | `ai_reviews_exhausted` | Monthly AI review limit reached |
| `403` | `feature_not_available` | AI Reviews not available on your plan |
| `404` | `crawl_not_found` | Crawl doesn't exist |
| `404` | `review_not_found` | Review doesn't exist |

### Limit Exceeded Response

```json
{
  "error": {
    "code": "ai_reviews_exhausted",
    "message": "You have used all 5 AI reviews for this month. Your allocation resets on December 15, 2024.",
    "used": 5,
    "limit": 5,
    "resets_at": "2024-12-15T00:00:00Z",
    "upgrade_url": "https://app.linkhealthmonitor.com/billing"
  }
}
```
