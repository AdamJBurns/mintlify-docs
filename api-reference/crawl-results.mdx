## Get Crawl Results

Retrieve detailed results from a completed crawl.

<ParamField header="Authorization" type="string" required>
  Bearer token with your API key
</ParamField>

<ParamField path="crawlId" type="string" required>
  The UUID of the completed crawl
</ParamField>

<ParamField query="type" type="string">
  Filter by result type: `all`, `errors`, `redirects`, `seo`, `success`
</ParamField>

<ParamField query="status" type="string">
  Filter by HTTP status: `2xx`, `3xx`, `4xx`, `5xx`
</ParamField>

<ParamField query="page" type="integer" default="1">
  Page number for pagination
</ParamField>

<ParamField query="limit" type="integer" default="100">
  Results per page (max: 500)
</ParamField>

### Request

```bash
curl -X GET \
  "https://api.linkhealthmonitor.com/v1/crawl/456e4567-e89b-12d3-a456-426614174100/results?type=errors&limit=50" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

### Response

<ResponseField name="crawlId" type="string">
  The crawl identifier
</ResponseField>

<ResponseField name="domain" type="string">
  The domain that was crawled
</ResponseField>

<ResponseField name="summary" type="object">
  Crawl summary statistics
  
  <Expandable title="Summary">
    <ResponseField name="health_score" type="integer">
      Overall health score (0-100)
    </ResponseField>
    <ResponseField name="total_urls" type="integer">
      Total URLs checked
    </ResponseField>
    <ResponseField name="broken_links" type="integer">
      Count of 4xx/5xx errors
    </ResponseField>
    <ResponseField name="redirects" type="integer">
      Count of redirects
    </ResponseField>
    <ResponseField name="seo_issues" type="integer">
      Count of SEO problems
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="results" type="array">
  Array of URL results
  
  <Expandable title="Result Object">
    <ResponseField name="url" type="string">
      The checked URL
    </ResponseField>
    <ResponseField name="status_code" type="integer">
      HTTP status code (or 0 for connection errors)
    </ResponseField>
    <ResponseField name="status_type" type="string">
      Category: `success`, `redirect`, `client_error`, `server_error`, `timeout`
    </ResponseField>
    <ResponseField name="response_time" type="integer">
      Response time in milliseconds
    </ResponseField>
    <ResponseField name="found_on" type="array">
      Array of pages containing this link
    </ResponseField>
    <ResponseField name="anchor_text" type="string">
      The link's anchor text (if applicable)
    </ResponseField>
    <ResponseField name="seo_issues" type="array">
      SEO problems found on this page
    </ResponseField>
  </Expandable>
</ResponseField>

## Get All URLs

Retrieve all checked URLs (not just errors):

```bash
curl -X GET \
  "https://api.linkhealthmonitor.com/v1/crawl/456e4567-e89b-12d3-a456-426614174100/results?type=all" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```bash

  "https://api.linkhealthmonitor.com/v1/crawl/456e4567-e89b-12d3-a456-426614174100/results?type=redirects" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## Export Results

Export results in CSV format:

```
GET /v1/crawl/{crawlId}/export
```

<ParamField query="format" type="string" default="csv">
  Export format: `csv` or `json`
</ParamField>

<ParamField query="type" type="string" default="all">
  What to export: `all`, `errors`, `redirects`, `seo`
</ParamField>

### Request

```bash
curl -X GET \
  "https://api.linkhealthmonitor.com/v1/crawl/456e4567-e89b-12d3-a456-426614174100/export?format=csv&type=errors" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -o broken-links.csv
```

### CSV Response

```csv
url,status_code,status_type,response_time,found_on,anchor_text,first_discovered
https://example.com/old-page,404,client_error,234,"https://example.com/;https://example.com/about",Learn more,2024-11-15T10:00:00Z
https://example.com/api/status,500,server_error,1523,https://example.com/developers,API Status,2024-12-02T15:01:00Z
```

## Error Responses

| Status | Code | Description |
|--------|------|-------------|
| `404` | `crawl_not_found` | Crawl doesn't exist |
| `400` | `crawl_not_complete` | Crawl is still running |
| `400` | `invalid_filter` | Invalid type or status filter |
