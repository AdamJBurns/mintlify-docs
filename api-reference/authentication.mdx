## Overview

The SEO Crawler API uses API keys for authentication. All API requests must include a valid API key in the request header.

<Info>
  **API Access**: Available on Pro and Agency plans only.
</Info>

## Getting Your API Key

<Steps>
  <Step title="Open API Settings">
    Navigate to **Settings** → **API** in your dashboard.
  </Step>
  
  <Step title="Generate API Key">
    Click **Generate API Key** to create a new key.
    
    <Warning>
      Your API key is only shown once. Copy and store it securely—you won't be able to view it again.
    </Warning>
  </Step>
  
  <Step title="Name Your Key">
    Give your key a descriptive name (e.g., "Production Server", "CI/CD Pipeline").
  </Step>
  
  <Step title="Copy and Store">
    Copy the key and store it in a secure location like a password manager or secrets vault.
  </Step>
</Steps>

## Using Your API Key

Include your API key in the `Authorization` header of every request:

```bash
curl -X GET \
  https://api.seocrawler.app/v1/domains \
  -H "Authorization: Bearer YOUR_API_KEY"
```

### Header Format

| Header | Value |
|--------|-------|
| `Authorization` | `Bearer YOUR_API_KEY` |

### Revoking Keys

To revoke an API key:

1. Go to **Settings** → **API**
2. Find the key to revoke
3. Click **Revoke**
4. Confirm revocation

<Warning>
  Revoking a key is immediate and permanent. Any applications using that key will stop working immediately.
</Warning>

## Rate Limits

API requests are rate limited to ensure fair usage:

| Plan | Requests per Minute | Requests per Day |
|------|---------------------|------------------|
| Pro | 60 | 10,000 |
| Agency | 300 | 100,000 |

### Rate Limit Headers

Every response includes rate limit information:

```
X-RateLimit-Limit: 60
X-RateLimit-Remaining: 45
X-RateLimit-Reset: 1701532800
```

| Header | Description |
|--------|-------------|
| `X-RateLimit-Limit` | Max requests per minute |
| `X-RateLimit-Remaining` | Remaining requests this minute |
| `X-RateLimit-Reset` | Unix timestamp when limit resets |

### Handling Rate Limits

When rate limited, you'll receive a `429 Too Many Requests` response:

```json
{
  "error": {
    "code": "rate_limit_exceeded",
    "message": "Rate limit exceeded. Please retry after 45 seconds.",
    "retry_after": 45
  }
}
```

Implement exponential backoff:

```javascript
async function apiRequest(url, options, maxRetries = 3) {
  for (let i = 0; i < maxRetries; i++) {
    const response = await fetch(url, options);
    
    if (response.status === 429) {
      const retryAfter = response.headers.get('Retry-After') || 60;
      await sleep(retryAfter * 1000);
      continue;
    }
    
    return response;
  }
  throw new Error('Max retries exceeded');
}
```

## Error Responses

### Authentication Errors

| Status | Code | Description |
|--------|------|-------------|
| `401` | `invalid_api_key` | API key is invalid or malformed |
| `401` | `expired_api_key` | API key has been revoked |
| `403` | `insufficient_permissions` | Key doesn't have required permissions |
| `403` | `plan_required` | Feature requires higher plan |

### Example Error Response

```json
{
  "error": {
    "code": "invalid_api_key",
    "message": "The provided API key is invalid.",
    "documentation_url": "https://docs.seocrawler.app/api-reference/authentication"
  }
}
```

## Security Best Practices

<CardGroup cols={2}>
  <Card title="Never Expose Keys" icon="eye-slash">
    Don't commit API keys to version control or expose them in client-side code.
  </Card>
  <Card title="Use Environment Variables" icon="server">
    Store keys in environment variables, not in code.
  </Card>
  <Card title="Rotate Regularly" icon="rotate">
    Rotate API keys periodically, especially after team member changes.
  </Card>
  <Card title="Use Separate Keys" icon="layer-group">
    Use different keys for different environments/applications.
  </Card>
</CardGroup>

### Environment Variables

```bash
# .env file (never commit this!)
SEO_CRAWLER_API_KEY=seo_live_abc123xyz789

# Usage in code
const apiKey = process.env.SEO_CRAWLER_API_KEY;
```

### Storing Keys Securely

Recommended storage options:

| Option | Best For |
|--------|----------|
| **Environment Variables** | Server applications |
| **Secrets Manager** | Cloud deployments (AWS, GCP, Azure) |
| **Vault** | Enterprise security requirements |
| **CI/CD Secrets** | Pipeline integrations |
