## Overview

SSL/TLS certificates are essential for website security and user trust. An expired certificate triggers browser warnings that drive away visitors. Link Health Monitor monitors your SSL certificates and alerts you before expiration.

<Info>
  **Available on**: All plans (Free, Solo, Pro, Agency)
</Info>

## What We Check

### Certificate Validity

| Check | Description |
|-------|-------------|
| **Expiration Date** | When the certificate expires |
| **Days Remaining** | Countdown to expiration |
| **Valid Chain** | Certificate chain is properly configured |
| **Hostname Match** | Certificate matches the domain |
| **Not Revoked** | Certificate hasn't been revoked |

### Certificate Details

| Field | Description |
|-------|-------------|
| **Issuer** | Certificate Authority (Let's Encrypt, DigiCert, etc.) |
| **Subject** | Domain the certificate covers |
| **Valid From** | Start date of validity |
| **Valid To** | Expiration date |
| **Algorithm** | Encryption algorithm (RSA, ECDSA) |
| **Key Size** | Encryption strength (2048-bit, etc.) |

## Why SSL Matters

### Security

- **Encryption**: Protects data in transit between users and your server
- **Authentication**: Verifies your site's identity to visitors
- **Integrity**: Ensures data isn't modified during transmission

### Trust & UX

- **Browser Indicators**: Green padlock builds user confidence
- **Warning Avoidance**: Prevents scary browser security warnings
- **Form Submissions**: Users trust SSL-protected forms with sensitive data

### SEO

- **Ranking Factor**: Google prioritizes HTTPS sites
- **Referral Data**: HTTPS preserves referrer information
- **Required Features**: Many modern web APIs require HTTPS

## Common Issues

### Expiring Certificate

<Tabs>
  <Tab title="Let's Encrypt">
    Let's Encrypt certificates expire every 90 days. Use automated renewal:
    
    ```bash
    # Test renewal
    sudo certbot renew --dry-run
    
    # Set up automatic renewal cron job
    0 0 * * * /usr/bin/certbot renew --quiet
    ```
    
    <Tip>
      Enable auto-renewal to never worry about Let's Encrypt expiration again.
    </Tip>
  </Tab>
  
  <Tab title="Commercial CAs">
    For DigiCert, Comodo, etc.:
    
    1. Log into your certificate provider dashboard
    2. Renew the certificate (often starts 30 days before expiration)
    3. Download the new certificate
    4. Install on your server
    5. Restart your web server
  </Tab>
  
  <Tab title="Cloud Providers">
    AWS, Cloudflare, and other cloud providers can manage certificates:
    
    **AWS Certificate Manager**:
    - Certificates auto-renew when used with AWS services
    
    **Cloudflare**:
    - Universal SSL auto-renews
    - Advanced certificates managed in dashboard
  </Tab>
</Tabs>

### Hostname Mismatch

<Note>
  This occurs when your certificate doesn't cover the domain being accessed.
</Note>

**Common causes**:
- Certificate for `example.com` but site accessed via `www.example.com`
- Certificate for main domain but subdomain uses same server
- Wildcard certificate (`*.example.com`) doesn't cover root domain

**Solutions**:
- Get a certificate that covers all needed domains (SAN certificate)
- Use a wildcard certificate AND a root domain certificate
- Redirect all traffic to the primary domain

### Incomplete Certificate Chain

Browsers need the complete certificate chain to validate trust:


If the intermediate is missing:

```bash
# Check your chain
openssl s_client -connect example.com:443 -servername example.com

# Combine certificates properly
cat your_domain.crt intermediate.crt > fullchain.crt
```

### Mixed Content Warnings

Even with valid SSL, mixed content (HTTP resources on HTTPS pages) triggers warnings:

```html
<!-- Bad: HTTP resource on HTTPS page -->
<img src="http://example.com/image.jpg">

<!-- Good: HTTPS or protocol-relative -->
<img src="https://example.com/image.jpg">
<img src="//example.com/image.jpg">
```

## Enabling SSL Monitoring

<Steps>
  <Step title="Start a Crawl">
    Navigate to your domain and click **Start Crawl**.
  </Step>
  
  <Step title="Enable SSL Check">
    In scan options, enable **SSL Certificate Check**.
  </Step>
  
  <Step title="Review Results">
    After the crawl, check the SSL section for certificate status.
  </Step>
  
  <Step title="Set Up Alerts">
    Configure email notifications for expiration warnings (Pro/Agency).
  </Step>
</Steps>
