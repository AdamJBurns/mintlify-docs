## Overview

AI Review uses advanced language models to analyze your crawl results and provide actionable recommendations. Instead of simply listing issues, AI Review prioritizes fixes by impact, explains why they matter, and provides ready-to-implement code.

<Info>
  **Available on**: Pro (5 reviews/month) and Agency (10 reviews/month) plans
</Info>

## What AI Review Provides

### Health Score

An overall score (0-100) reflecting your website's health based on:
- Broken links and server errors
- Redirect efficiency
- SEO optimization
- Performance metrics
- SSL status

### Prioritized Issues

Issues ranked by:
- **Severity**: How much the issue hurts your site
- **Impact**: How many users/pages are affected
- **Effort**: How difficult the fix is

### Actionable Categories

| Category | Description |
|----------|-------------|
| **Critical Issues** | High-priority problems requiring immediate attention |
| **Quick Wins** | Easy fixes with significant positive impact |
| **Optimizations** | Improvements that boost performance and SEO |
| **Best Practices** | Recommendations for long-term site health |

### Code Fixes

Where applicable, AI Review provides:
- Specific code snippets to implement
- Before/after comparisons
- File locations or component references
- Implementation notes

## How It Works

<Steps>
  <Step title="Run a Crawl">
    Complete a crawl with your desired scan options enabled. More options = more data for analysis.
  </Step>
  
  <Step title="Request AI Review">
    After the crawl completes, click **Generate AI Review**. Reviews typically complete in 30-60 seconds.
  </Step>
  
  <Step title="Review Findings">
    AI Review presents findings in order of priority with explanations and code fixes.
  </Step>
  
  <Step title="Implement Fixes">
    Work through recommendations, implementing fixes and re-crawling to verify.
  </Step>
</Steps>

### Provide Context

When requesting a review, you can optionally provide context:

- **Site type**: E-commerce, blog, SaaS, etc.
- **Priority goals**: SEO, performance, accessibility
- **Recent changes**: New deployment, redesign, migration

### Implement Incrementally

<Tip>
  Don't try to fix everything at once. Work through critical issues first, then quick wins, then optimizations. Re-crawl and review after each batch to verify progress.
</Tip>

## AI Review vs Manual Analysis

| Aspect | Manual Analysis | AI Review |
|--------|-----------------|-----------|
| **Time** | Hours to days | 30-60 seconds |
| **Prioritization** | Requires expertise | Automatic by impact |
| **Code Fixes** | Research required | Ready to implement |
| **Explanations** | Varies by analyst | Consistent, clear |
| **Scalability** | Limited | Handles any site size |

## Understanding the Health Score

### Score Components

| Factor | Weight | Description |
|--------|--------|-------------|
| **Broken Links** | 25% | 4xx/5xx errors and timeouts |
| **Redirects** | 15% | Chain length and loops |
| **SEO Elements** | 25% | Titles, descriptions, H1s, alt text |
| **Performance** | 15% | Response times and page size |
| **SSL/Security** | 10% | Certificate status |
| **Accessibility** | 10% | Basic accessibility factors |

### Score Interpretation

| Score | Rating | Meaning |
|-------|--------|---------|
| 90-100 | Excellent | Site is well-optimized with minimal issues |
| 75-89 | Good | Some issues but fundamentals are solid |
| 50-74 | Fair | Multiple issues needing attention |
| 25-49 | Poor | Significant problems affecting users |
| 0-24 | Critical | Severe issues requiring immediate action |
